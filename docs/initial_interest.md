# MentorMind Product Specification and Behavioral Requirements

---

## User Onboarding and Interest Mapping

The application must understand the learner before generating content. This ensures that every lesson is contextually relevant.

* **Initial Data Collection** Implement a multi step onboarding flow after a user registers. This form must collect the user's current grade level and their primary subjects of interest like Mathematics or Computer Science. It should also ask about their specific learning struggles and long term academic goals.
* **Database Integration** Create a** **`user_profiles` table in PostgreSQL to store these attributes. Every time the** **`analyze-topics`endpoint is called, the backend must fetch this profile data to weight the AI suggestions toward the user's established interests.
* **Personalized Dashboard** The home screen should display a "Recommended for You" section. This section uses the user's profile to pre generate interesting lesson prompts instead of waiting for a manual search query.

---

## The Interactive Learning Experience

A lesson should be a conversation rather than a broadcast.

* **Pause and Ask Feature** Enhance the video player in** **`app/lessons/[id]/page.tsx` with a dynamic chat overlay. When a user pauses the video, they should be able to ask a question like "Can you explain that last part again?". The backend must receive the video timestamp to provide a contextually accurate response based on the lesson script.
* **Knowledge Checkpoints** Every generated lesson must include at least three interactive quiz questions at the end. These are generated by the** **`agentic.py` module during the initial creation process.
* **Performance Tracking** Save the results of these quizzes into a** **`student_performance` table. This data will allow the AI to identify which topics the student has mastered and which require a follow up lesson.

---

## Monetization and Usage Policies

To ensure sustainability, the application requires a robust credit and subscription system.

* **The Credit Economy** Define a credit as one full lesson generation. The** **`User` model must track a** **`credit_balance` and a** **`subscription_tier`.
* **Tiered Access**
  * **Free Tier:** Users receive 5 free credits every month. They are limited to text based queries and standard TTS voices.
  * **Pro Tier:** Users receive 60 credits per month. They gain access to multimodal ingestion like PDF and Audio uploads along with premium emotive voices.
  * **Elite Tier:** Designed for educators with 250 credits per month and the ability to export lessons to external learning management systems.
* **Enforcement Middleware** Create a middleware in the FastAPI backend that checks the user's credit balance before allowing a** **`celery_task` to start. If the balance is zero, the API must return a "Payment Required" status code to trigger the upgrade UI on the frontend.

---

## Global Language Consistency

The AI must respect the user's language choice across all generated content.

* **Content Lock** The** **`language` parameter from the frontend must be injected into every LLM system prompt. The instruction should explicitly state: "All titles, explanations, and exercises must be written entirely in the target language. Do not mix English and Chinese."
* **Voice Synchronization** The TTS engine must automatically switch to a native sounding voice based on the content language. A Chinese lesson should never be narrated by an English voice model.
