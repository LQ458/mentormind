# MentorMind 提升计划：以视频质量为核心

## 核心思路

目前平台最大的瓶颈是**视频质量**。视频质量的天花板，本质上取决于两件事：

1. **提示词（Prompt）的质量** — AI 的"剧本导演"。提示词写得越专业，生成的课程内容越符合真实教学逻辑。
2. **渲染引擎的表现力** — 动画流畅度、色彩美感、节奏感。

> **关键洞察：** 招一位真正的优秀老师来参与提示词设计，效果会远超纯技术优化。老师懂怎么讲，AI 才能学着怎么教。

---

## 一、应用修改部分

### 1. 教案 / 课程大纲结构

- 梳理常见学科（如高中数学、物理）的核心知识点体系
- 手写 5-10 个"教学案例"，明确每一节课应该：
  - 引入什么问题？
  - 讲解哪些核心概念？
  - 结尾做什么总结？
- 这些案例可以直接作为 Prompt 设计的参考材料和测试用例

### 2. 提示词调试与测试

- 对比不同提示词生成的视频，记录哪种风格更"像课堂"
- 收集用户反馈，迭代改进

### 3. 产品体验与 UI 打磨

- 评估现有用户引导流程（Onboarding）是否足够简洁有效
- 整理用户测试中观察到的交互问题

---

## 二、需要找老师帮助的部分

### 1. 提示词设计：最需要老师参与！

| 提示词文件                                                                                                         | 作用                   | 老师可以贡献什么                                         |
| ------------------------------------------------------------------------------------------------------------------ | ---------------------- | -------------------------------------------------------- |
| `lesson_syllabus.md`                                                                                             | 生成课程大纲           | 确认大纲是否符合教学逻辑；什么应该先教，什么后教         |
| [storyboard_builder.md](file:///Users/LeoQin/Documents/GitHub/mentormind/backend/prompts/video/storyboard_builder.md) | 生成每一幕的可视化脚本 | 提供"课堂上老师会怎么画图讲解"的真实示例                 |
| [mentor_chat](file:///Users/LeoQin/Documents/GitHub/mentormind/backend/server.py#145-168) 提示词                      | AI 导师与学生对话      | 提供真实的因材施教对话示例；老师"探问学生理解程度"的话术 |

**建议方式：** 约教该学科的老师进行 1-2 次访谈（1小时），问他们：

- 讲这个概念时，你会在黑板上画什么？
- 学生最常在哪里卡住？你怎么引导？
- 你一节课的"节奏"是什么样的？

把这些范式写进提示词，效果会飞升。

### 2. 课程测试与校对

- 请老师观看 AI 生成的视频，指出讲错的、讲浅的、顺序不对的地方
- 每科目各招募 1 位老师（可以是在校大学生助教），作为"内容顾问"

## 三、分阶段推进计划

### 阶段一：提示词提升（1-2 个月）

- [ ] 自己整理 5 份教学案例模板（从高中数学/物理开始）
- [ ] 找 1-2 位学科老师进行访谈，提炼提示词核心原则
- [ ] 重写 [storyboard_builder.md](file:///Users/LeoQin/Documents/GitHub/mentormind/backend/prompts/video/storyboard_builder.md) 和 `lesson_syllabus.md`，使 AI 生成更像"真实课堂"
- [ ] 生成 10 个测试视频，对比修改前后的效果

### 阶段二：视频美化（1 个月）

- [ ] 实现深色主题 + 渐进式动画
- [ ] 实现 LaTeX 清洗护栏，杜绝"红色错误画面"
- [ ] 上线对话式诊断 Onboarding（替代当前多项选择问卷）

### 阶段三：增长与变现（3 个月后）

- [ ] 集成视频播放事件追踪（观看进度、完播率）
- [ ] 集成 Stripe 订阅/代币体系
- [ ] 扩展学科范围（化学、英语写作等）

## 四、关于找老师的建议

**在哪找：**

- 在读研究生 / 师范类院校同学（愿意参与新技术项目）
- 微信教师群或知乎"教育从业者"社群
- 你的校友资源（本科/高中老师）

**怎么合作：**

- 不需要正式雇佣，项目顾问形式即可
- 可以给小额咨询报酬，或者产品内测资格 + 署名
- 每次合作聚焦一个学科，产出"该学科的提示词手册"
