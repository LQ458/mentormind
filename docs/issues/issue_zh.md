# MentorMind 平台

## “低劣视频质量”问题

生成引人入胜、高质量的程序化视频（使用 Manim + AI）在计算和创意上都非常困难。目前，由于以下流水线限制，生成的视频缺乏打磨：

### A. 视觉单调（排版停滞）

- **症状：** AI 严重偏向于使用 `callout_card` 或 `two_column` 排版，基本上将视频当作 PowerPoint 演示文稿，而不是动态黑板。
- **根本原因：** `storyboard_builder` 提示词没有强制 AI 足够频繁地采用动态 `<transform>` 或 `<draw_shape>` 动作。它默认使用安全的静态文本。

### B. 数学渲染失败（“红色错误”画面）

- **症状：** 涉及复杂数学的场景在内部渲染失败，或者输出损坏的字符，导致视频流水线直接丢弃该场景。
- **根本原因：** 大语言模型（LLM）偶尔会生成原生 LaTeX（比如 `\begin{align}`），除非明确格式化，否则 Manim 默认的 `MathTex` 环境很难处理它。

### C. 节奏与信息密度

- **症状：** 一个画面可能有 200 个词的旁白，但屏幕上只有 5 个词，让视频在视觉上感觉“卡住了”。
- **根本原因：** 时长映射逻辑纯粹是数学计算（`总字数 / 2.1`），完全没有考虑视觉认知负荷。文本是累积地丢到屏幕上的，而不是渐进式地产生动画（例如，逐个要点显示）。

### D. 色彩理论与美学

- **症状：** 默认的 Manim 使用生硬的黑色背景和普通的白/蓝色文本，与周围的 Web UI 相比显得非常粗糙和未经打磨。
- **根本原因：** 我们在 `ManimService` 中没有注入主题设计系统。

### 视频打磨与美化

1. **主题升级：** 全面改造 `ManimService`，采用优美统一的调色板（例如，Catppuccin Macchiato `#1E1E2E`）和定制的现代字体。
2. **生成式节奏：** 更新引擎以自动串联动画（例如，`Write` -> `Wait` -> `FadeIn`），使文本在旁白说话时感觉“活”起来。
3. **LaTeX 清洗护栏：** 构建一个健壮的基于正则表达式的清洗器，在 LLM 输出进入 Manim 编译器之前，温和地去除不支持的环境。

### 智能用户引导

- **问题：** 当前通用的多项选择题引导问卷限制了 AI 准确评估学生水平的能力。
- **解决方案（对话式诊断）：** 将引导流程转变为 3 轮互动的对话式诊断，由 AI 导师要求学生解决一个真实的简短问题。然后 AI 会根据他们实际的表现来提取真实的基准知识，而不仅仅是依靠他们自我报告的答案。

### 增长与数据分析

- **问题：** 课程数据分析和定价目前完全没有实现。
- **解决方案：** 通过监控视频参与度（观看百分比）和测验完成率来追踪平台价值。集成 Stripe，采用代币化或分级的订阅模式，以抵消高昂的计算推理成本。
