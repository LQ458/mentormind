# MentorMind 功能说明文档

> AI 驱动的双语（中/英）个性化教学平台。后端基于 FastAPI + Celery + PostgreSQL + Redis，前端基于 Next.js 14 + TypeScript + Tailwind。
> 核心特色：**流式白板**（受豆包爱学启发）、**多阶段 AI 导师对话**、**Manim 数学动画自动生成**。

## 1. 用户系统与认证

### 1.1 认证机制
- **Clerk JWT 认证**：通过 RS256 签名 + JWKS 公钥验证（`auth.py:decode_token()`）。
- **自动用户预置**：首次登录时通过 `clerk_id_to_uuid()` 生成确定性 UUID5，自动同步到 PostgreSQL。
- **依赖注入**：`get_current_user()` 作为 FastAPI 依赖项，所有受保护端点统一鉴权。

### 1.2 用户档案字段
- **基本信息**：username、email、full_name、is_premium、language_preference（zh / en）
- **学习画像（UserProfile）**：
  - grade_level（学段：初中、高中、本科、研究生、职场）
  - subject_interests（学科兴趣，多选）
  - current_challenges（当前学习困难）
  - long_term_goals（长期学习目标）
  - preferred_learning_style（学习风格：视觉型、实践优先、概念优先、对话式）
  - weekly_study_hours（每周学习时长）
  - onboarding_completed（是否完成首次问卷）
- **学科熟练度（SubjectProficiency）**：subject、proficiency_level、confidence、last_updated
- **学习表现（StudentPerformance）**：assessment_type、score、strengths、struggles、reflection

### 1.3 兴趣画像问卷（InterestProfileQuiz）
- 首次创建课程时引导用户填写。
- 多步骤分步呈现，已填项高亮蓝色边框。
- 加载状态使用骨架屏占位动画。

### 1.4 订阅与计费
- **三档计划**：Basic / Professional / Enterprise
- **Stripe 集成**：
  - `/billing/create-checkout-session`：发起支付
  - `/billing/webhook`：接收订阅事件并更新 is_premium
- **额度追踪**：每月已用课程数、剩余配额、累计花费

## 2. 课程生成管线（核心管线）

### 2.1 入口端点
`POST /create-class` 接收主题、学段、语言、时长、风格预设，立即返回 job_id 并入队 Celery 任务。

### 2.2 六阶段流程
| 阶段 | 描述 | 核心模块 |
|------|------|----------|
| 1. 认知处理 | 提取学习目标、构建大纲 | `core/modules/cognitive.py` |
| 2. 内容生成 | DeepSeek LLM 生成课程主体 | `services/api_client.py` |
| 3. 视频脚本 | 生成 Manim 场景代码 + 旁白 | `core/modules/video_scripting.py` |
| 4. 渲染 | Manim 并行渲染（最多 6 并发，semaphore 控制） | `core/rendering/manim_renderer.py` |
| 5. TTS 合成 | 各旁白段落生成音频 | `services/tts/service.py` |
| 6. 后处理 | 视频拼接、音频时间对齐、缩略图 | `core/modules/output.py` |

### 2.3 容错机制
- **LLM 自修复**：渲染失败时调用 DeepSeek 修复 Manim 语法错误，最多重试 3 次（`_fix_code_with_llm()`）。
- **降级链**：HeyGen → 本地 Manim；SiliconFlow TTS → edge-tts。
- **熔断器**（CircuitBreaker）：closed → open → half_open，外部 API 故障时快速失败。

### 2.4 进度追踪
- Celery 任务向 Redis 实时写入进度：`hset(f"job:{job_id}", mapping=progress)`
- **双通道客户端**：
  - `GET /job-status/{job_id}`：轮询接口
  - `GET /job-stream/{job_id}`：Server-Sent Events 流式推送
- 前端策略：先尝试 SSE，12 秒未通则降级到 2 秒间隔轮询。

## 3. AI 导师多阶段对话

### 3.1 状态机
`MentorAgent` 实现四阶段（MentorStage）：

| 阶段 | 行为 | 文件位置 |
|------|------|----------|
| opening | 学科识别 + 开场问候 | `core/modules/mentor.py:_handle_opening()` |
| diagnostic | 自适应诊断（最多 3 轮） | `_handle_diagnostic()` |
| roadmap | 生成学习路线图 + 大纲 | `_handle_roadmap()` |
| co_creation | 协作式问题求解 | `_handle_co_creation()` |

### 3.2 端点
`POST /mentor/chat` 接收 `MentorChatRequest`，返回 `MentorChatResponse`，包含：
- `text`：当前回复
- `stage`：当前所处阶段
- `thinking_process`：AI 的"思考过程"（前端蓝色斜体框展示）
- `proposed_syllabus`：路线图阶段返回大纲
- `diagnostic_question`：诊断阶段返回当前题目

### 3.3 流式响应
- DeepSeek `chat_completion_stream()` 流式输出 token。
- 前端逐字呈现，提升对话感。

## 4. 流式白板系统（核心特色）

> 项目记忆显示：**Phase A + B 已完成**，灵感来自字节跳动豆包爱学。

### 4.1 元素类型（14 种）
- **文本类**：title（标题）、text_block（段落）、definition_box（定义框，黄底）、theorem_box（定理框，蓝底）
- **数学类**：equation（KaTeX 公式）、graph（D3 可视化）
- **代码与表格**：code_block（语法高亮）、table（Markdown 表）
- **图形类**：shape（SVG 矩形/圆/三角）、arrow（连接箭头）、image（懒加载图片）
- **结构类**：step_list（编号步骤）、highlight（高亮标注）、transform（CSS 变换）

### 4.2 动画与样式
- **入场动画**：fade_in（淡入）、write（带模糊的书写）、grow（缩放放大）、slide_in（侧滑）、none
- **统一时长**：0.35 秒，easeOut 缓动
- **颜色系统**：accent / heading / text / green / mauve / yellow / red
- **尺寸系统**：small / medium / large / xlarge
- **背景主题**：dark_board（默认）、light_board、grid（网格）、plain
- **布局**：full_canvas、split_left_right、split_top_bottom、focus_center

### 4.3 MCP 协议工具
通过 `mcp/board_server.py` 提供符合 MCP 规范的工具调用接口，由 LLM 在生成课程时调度：

| 工具 | 功能 |
|------|------|
| board_create | 初始化白板（标题、布局、背景） |
| board_add_element | 添加可视化元素 + 可选旁白 |
| board_update_element | 修改/高亮/移动/动画/移除元素 |
| board_clear | 清空整块或区域 |
| board_focus | 设置焦点区域 |
| board_narrate | 排队旁白文本进行语音合成 |
| board_checkpoint | 创建理解检查点 |
| board_transition | 状态间过渡动画 |

工具调用经 `ToolCallValidator`（JSON Schema 校验）后才执行，防止 LLM 输出无效参数。

### 4.4 实时流式传输
- **WebSocket 端点**：`/ws/board/{session_id}`，JWT query 鉴权。
- **状态管理**：`BoardStateManager` 维护内存状态 + 完整事件日志（用于断线重连）。
- **支持指令**：暂停 / 恢复 / 跳过 / 重新讲解。
- **事件类型**：board_add_element、board_update_element、narration、checkpoint、summary_ready、error。

### 4.5 TTS 同步
- `BoardTTSSync` 协调音频时长与白板动画时序。
- TTS 段落附带 duration_ms 元数据，客户端依此安排元素出场节奏。

### 4.6 理解检查点
- `ComprehensionCheckpoint`：中途弹出题目打断学习。
- 学生作答后调用 `/board/checkpoint-response`，AI 据此调整后续节奏。

### 4.7 会话总结
- `core/board/summarizer.py:summarize_session()` 将事件日志 + 旁白拼成 markdown 摘要。
- `/board/session/{session_id}/summary` 触发生成，前端在右侧抽屉滑入展示。

### 4.8 再讲一遍
`POST /board/explain-differently` 针对学生不理解的板块，换风格重新讲解（详细/简洁/类比/类比生活）。

### 4.9 限流
- 每用户最多 5 个并发会话；全局最多 100 个。

## 5. 学习内容管理

### 5.1 课程库
- `GET /lessons`：分页 / 筛选 / 排序
- `GET /lessons/{id}`：完整结构（含 objectives、resources、exercises）
- `DELETE`：软删除（status = deleted）+ 批量删除
- 导出 PDF（前端调用 jsPDF）

### 5.2 学习计划（Study Plan）
- `POST /study-plan/chat`：多轮对话设计计划
- `POST /study-plan/create`：保存计划
- 计划由若干 Unit 组成，每个 Unit 可独立生成课程或板课
- 状态：draft / active / completed / archived

### 5.3 复习队列（间隔重复）
- 基于 **F5 算法** 自动调度复习时机
- `GET /users/me/review-queue`：返回待复习课程
- 每晚 Celery beat 任务（`sync_proactive_notifications_task`）刷新提醒

### 5.4 五种交互式练习
| 类型 | 端点 | 用途 |
|------|------|------|
| 研讨会 | `/seminar` | 多智能体讨论一道题 |
| 模拟 | `/simulation` | 应用场景化练习 |
| 口述答辩 | `/oral-defense` | 反思式口述检验 |
| 记忆挑战 | `/memory-challenge` | 间隔重复测试 |
| 故意出错 | `/deliberate-error` | 主动找出含错的讲解 |

### 5.5 主动通知
- `ProactiveNotification` 模型：复习提醒、推荐、里程碑、警告
- `/users/me/notifications` 列表 + 标记已读 / 关闭

### 5.6 视频参与度埋点
- `/users/me/lessons/{lesson_id}/video-engagement` 上报观看片段
- 用于热力图分析与质量评估

### 5.7 媒体上下文
- `UserMediaContext`：用户上传过的音频/图片均缓存可复用
- `/user/storage-usage`：配额查询

## 6. 多模态输入

### 6.1 音频转写（ASR）
- **本地 FunASR**（Paraformer-zh）：中文最优
- **本地 Whisper**：英文与多语言
- **自动语言检测**：`_detect_audio_language()`
- 音频预处理：统一为 16 kHz 单声道 WAV
- 端点：`POST /ingest/audio`（异步入队）
- Celery 任务：`transcribe_audio_task`（heavy_ml 队列）

### 6.2 图像 OCR
- **PaddleOCR**：中文文字识别强项
- 优先调用微服务（`PADDLE_OCR_ENDPOINT`），失败降级本地模型
- 端点：`POST /ingest/image` + `POST /study-plan/ask-ai`（图问答）
- Celery 任务：`ocr_image_task`

### 6.3 TTS 文字合成
- **中文**：SiliconFlow TTS（首选）→ edge-tts（无 API key 降级）
- **英文**：OpenAI TTS
- 端点：`GET /voices`（列出可用音色）
- 批量合成：`TTSSynthesizer` 用于课程旁白

## 9. 异步任务系统（Celery）

### 9.1 配置
- **Broker**：Redis（`CELERY_BROKER_URL`）
- **三个隔离队列**：
  - `orchestration`：课程编排
  - `rendering`：Manim 渲染
  - `heavy_ml`：ASR / OCR
- **解决 PyTorch / Whisper SIGSEGV**：使用 solo pool（不 fork）
- **任务时限**：硬限制 30 分钟，软限制 25 分钟
- `acks_late = True`：crash 时不丢消息

### 9.2 任务清单
| 任务 | 队列 | 触发 |
|------|------|------|
| sync_proactive_notifications_task | orchestration | beat 每 24 小时 |
| create_class_video_task | orchestration | 用户创建课程 |
| transcript_to_lesson_task | orchestration | 转录 → 课程 |
| transcribe_audio_task | heavy_ml | 上传音频 |
| ocr_image_task | heavy_ml | 上传图片 |
| generate_unit_content_task | orchestration | 学习计划单元生成 |
| regenerate_board_segment_task | orchestration | 板课段重生成 |
| rollup_proficiency_task | orchestration | 学科熟练度滚动 |

## 11. 双语支持

### 11.1 语言参数
所有创建课程 / 板课接口都接受 `language: "en" | "zh"`，决定：
- ASR 模型选择（Whisper / FunASR）
- TTS 服务选择（OpenAI / SiliconFlow + edge-tts）
- 提示词模板选择

### 11.2 提示词系统
- `prompts/loader.py:render_prompt()`：模板渲染 + 变量替换
- `prompts/language/language_instruction.md`：语言特定指令前缀
- 所有 LLM 请求都自动注入语言指令

### 11.3 中文 NLP 库
- `zhconv`：繁简转换
- `jieba`：分词
- `cn2an`：数字与汉字互转
- `pynlpir`：中文 NLP

### 11.4 前端 i18n
- `web/app/lib/translations.ts`：200+ 翻译键
- `t(path, language, variables)`：支持变量替换
- **UI 语言** 与 **内容语言** 独立切换（分别存 localStorage）
- 自动设置 `document.documentElement.lang`

## 14. 现有 UX 细节

### 14.1 视觉与排版
- **时间感问候**：早上好 / 下午好 / 晚上好 + 名字 + emoji
- **暗色白板主题**：slate-950 背景 + 半透明卡片 + 高亮发光（`shadow-[0_0_28px_rgba(253,224,71,0.35)]`）
- **状态在线指示**：`bg-yellow-400 animate-pulse` 表示服务在线

### 14.2 动画 & 反馈
- **白板元素入场**：framer-motion，0.35 秒 easeOut，按元素类型选动画
- **总结面板滑入**：从右侧 0.25 秒 tween
- **进度条平滑**：300 ms 趋近真实百分比，避免跳变
- **聊天加载点**：3 个 `animate-bounce` 错时 0 / 0.1 / 0.2 秒
- **路线图章节圆圈悬停**：`group-hover:scale-110`

### 14.3 自动滚动
- 白板自动滚到最新元素
- 检测用户上滑超过 80px 后停止自动滚动，避免打扰阅读
- 用户滚回底部或课程恢复后再次启用

### 14.4 交互细节
- **AI Testing Mode 黄色高亮开关**：跳过诊断直接生成（开发用）
- **诊断进度指示**：「基础诊断 1/3」+ 蓝点状态条
- **Enter 发送 / Shift+Enter 换行**
- **音频/图片上传异步轮询**：5 秒间隔，最长 5 分钟
- **失败原因映射**：网络 / API key / 视频环境 → 可读中文提示
- **白板暂停浮层**：右上角琥珀色「Paused — scroll to review」

### 14.5 课程设计开关组
创建课程时可启用：
- showThinkingPath：显示知识图谱
- enableSeminar：研讨会模式
- enableSimulation：场景模拟
- enableOralDefense：口述答辩
- addDeliberateError：故意出错
- personalAnchor：个人锚点
- verbosity：旁白详略（compact / standard / thorough）

### 14.6 可访问性
- 所有图标按钮 `aria-label`
- 语言切换 `role="group" aria-label="Language"`
- 检查点 `role="dialog"`
- 侧栏当前页 `aria-current="page"`

## 附录 B：典型用户旅程

1. **首次注册**：Clerk 登录 → 自动创建 user + UUID5 → 跳转 dashboard。
2. **首次创建课程**：`/create` 触发兴趣画像问卷 → 保存 UserProfile → 进入聊天阶段。
3. **多阶段对话**：opening → diagnostic（最多 3 轮）→ roadmap → 用户确认 → 进入生成阶段。
4. **课程生成**：调用 `/create-class` 入队 Celery → SSE 接收实时进度（5 阶段管线进度条）→ 完成后跳转 `/lessons/[id]`。
5. **板课学习**：`/board/[sessionId]` WebSocket 连接 → 元素逐个出现 + 旁白同步 → 中途检查点 → 学完查看总结。
6. **复习**：每晚 Celery 生成主动通知 → dashboard 提示 → 进入复习队列做记忆挑战。
7. **进度追踪**：每节课后录入 performance → 累积到 SubjectProficiency → 影响后续课程难度推荐。
